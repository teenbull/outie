Это простой Python-скрипт, который собирает все текстовые файлы из папки проекта в один файл. Он читает правила из `.gitignore` и `.outignore`, чтобы понимать, какие файлы нужно пропустить.

Основная задача инструмента — упаковать исходный код в один файл, чтобы его было удобно передать нейросетям (LLM) одним куском. Удобно для случая когда ваш формат файлов не поддерживается в LLM или трудно закидывать отдельно десятки файлов из десяток папок.

## Особенности
* **Поддерживает правила исключений:** Читает `.gitignore` и `.outignore`.
* **Пропускает лишнее:** Автоматически игнорирует бинарные файлы (например, изображения или скомпилированный код), если их нельзя прочитать как обычный текст.
* **Без сторонних библиотек:** Просто кладем скрипт в папку и запускаем.

## Требования

* Python 3.6 или новее.

## Использование

1. Поместите файл `output.py` в папку вашего проекта.
2. Запустите скрипт любым способом (двойной клик, запуск из редактора или команда):

```bash
python output.py
```

Скрипт создаст файл `output.txt` в этой же папке. При этом он автоматически исключит из сборки самого себя и итоговый файл.

## Настройка

Если вам нужно скрыть дополнительные файлы от скрипта, а `.gitignore` для этого недостаточно, создайте файл `.outignore`. Он просто дополнит правила `.gitignore`. Для игнорирования файлов/папкок используются те же правила синтаксиса, что и в `.gitignore`:

Пример `.outignore`:
```text
package-lock.json
*.min.js
tests/
docs/
```
### Как игнорировать папки
`название_папки/` — Игнорирует папку с таким именем везде (на любом уровне вложенности). Слэш на конце означает «только папка».
Пример: `node_modules/` (проигнорирует и `/node_modules/`, и `/frontend/node_modules/`).
Важно: Если написать просто `build` (без слеша), скрипт проигнорирует и папку `build`, и файл с именем `build`. Поэтому для папок лучше всегда ставить / в конце.
`/название_папки/` — Игнорирует папку только в корне проекта.
Пример: `/venv/` (проигнорирует `/venv/`, но оставит `/backend/venv/`).

### Как игнорировать файлы
`имя_файла.ext` — Игнорирует файл с таким именем везде.
Пример: `secrets.json` (будет проигнорирован и в корне, и в подпапках).
`/имя_файла.ext` — Игнорирует файл только в корне проекта.
Пример: `/.env` (проигнорирует `/.env`, но оставит `/docker/.env`).
`*.ext` — Игнорирует все файлы с таким расширением.
Пример: `*.log` (проигнорирует `error.log`, `app.log` в любой папке).
`название_*` — Игнорирует файлы по началу имени.
Пример: `config_*.yml` (проигнорирует `config_dev.yml`, `config_prod.yml`).

### Спецсимволы (Маски)

`*` (звездочка) — заменяет любое количество символов.
`* backup *` — проигнорирует всё, где есть слово "backup" (`my_backup.zip`,` backup_folder/`).
`?` (знак вопроса) — заменяет ровно один символ.
`test?.py` — проигнорирует `test1.py`, `test2.py`, но не `test10.py`.

### Комментарии и пустые строки

Строки, начинающиеся с #, считаются комментариями и игнорируются.
Пустые строки нужны просто для красоты и удобства чтения, они тоже игнорируются.


## Ограничения

1. **Нет поддержки двойных звездочек (`**`):** Правила вида `src/**/*.txt` не будут работать так, как в Git. Скрипт воспринимает их как обычные одинарные звездочки.
2. **Нет отмены игнорирования (`!`):** Правила, начинающиеся с восклицательного знака (используются для возврата ранее исключенного файла), не поддерживаются.
3. **Лимит памяти:** Скрипт читает файлы целиком в оперативную память перед тем, как сохранить их. Если в проекте есть огромные текстовые файлы (например, текстовый лог на несколько гигабайт), добавьте их в `.outignore`, иначе скрипт может упасть с ошибкой нехватки памяти, хотя это маловероятно.
